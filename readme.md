# 📘 Kaufland POSTING

Сервис для взаимодействия с KauflandAPI, сервис умеет удалять выставлять, проверять и обновлять цены с фронтендом
---

# Сервис написан на Django + Postgres + Docker(Nginx)
## Структура проекта
### Папка: kaufland_API
- asgi.py - Настройка uvicorn для websocket
- settings.py - Настройки приложения
- urls.py - Включаем url адреса приложения и корневый url
---
### Папка: main_api
- migrations миграции(не нужны)
- src вся аутсорс логика
- src/controller - логика взаимодействия с API KAUFLAND
- src/ftp_src - логика взаимодействия 1 веб сервера и загрузки туда обработанных фотографий
- src/gpt - вся логика с обращением GPT для генерации description
- extracter.py вся логика для добавления описания товара из html файла (media/html)
- src/token_handler(не нужны)
- templates - Здесь хранится фронт
- consumers, context_processors, routing - Нужны для работы с WebSocket
- serializers - Несколько сериализаторов для работы с JSON и контроллерами JV, XL
- views - 2 основные View (MainOperationsView-Удаление, Чекер, изменение цен, UploadCollectionsViaJsonView - выставление коллекций по .json файлам)
### Папка: media - Обычная папка куда грузятся медиа файлы
### Папка: static - Нужные часть только css и js в js используется только main.js
### .env файл для локальной разработки
### .env.docker файл для докера
### nginx.conf настройка для внутреннего nginx для реверс прокси

## Особенности
### Мой сервис работает по JWT токенам
### Для взаимодействия по API данные меняются не сразу а спустя 30-60 минут, используется HMAC подпись для реквестов
### Для продукта существует его тело которое нельзя удалить это некий каркас и оффер это то что видят клиенты
### RateLimiting 111rps

## Код
#### Основная логика прописана в файле main_api/src/controller/kaufland_controller. Существует класс KauflandController который отвечает за взаимодействие с площадкой
#### Статичные функции которые не нужно трогать: _sign_request(Подпись по HMAC для валидного запроса), _get_headers(Функция которая подгоняет headers под хотелки площадки), _universal_request(Функция которая обрабатывает ошибки платформы, следит за лимитом реквестов и использует 5 методов необходимых для Kaufland)
#### Функции помощники которые помогают правильно оформить данные: get_exchange_rate(функция в которой хранятся курсы валют), format_price(функция которая из неровной цены делают красивую для продаж), status_processing(отслеживает статус для асинхронных тасков)
#### Остальные функции это рабочие функции описание описано в документации
